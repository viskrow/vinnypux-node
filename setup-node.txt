───────────────────────────────────────────────
 Установка ноды
───────────────────────────────────────────────

1. Создай ноду в панели Remnawave → Nodes → скопируй SECRET_KEY
2. Запусти одну команду на новом сервере:

───────────────────────────────────────────────
 Установка с GitHub (одна команда)
───────────────────────────────────────────────

# Selfsteal + Bridge нода (с реальным wildcard cert)
CF_Token="cf_xxxxx" bash <(curl -fsSL https://raw.githubusercontent.com/viskrow/vinnypux-node/main/setup-node.sh) \
  --with-bridges

# WL нода (CDN-схема, реальный wildcard cert не нужен)
bash <(curl -fsSL https://raw.githubusercontent.com/viskrow/vinnypux-node/main/setup-node.sh) \
  --with-wl

# Минимальная Reality-нода (с wildcard cert через Cloudflare)
CF_Token="cf_xxxxx" bash <(curl -fsSL https://raw.githubusercontent.com/viskrow/vinnypux-node/main/setup-node.sh) \
  --secret-key "SECRET_KEY_ИЗ_ПАНЕЛИ"

# Дополнительные флаги:
#   --cf-token "..."     Cloudflare API token для wildcard cert (или ENV CF_Token)
#                          Без него — placeholder cert (real клиенты не пойдут)
#   --no-update          пропустить apt upgrade
#   --ssh-port 2222      если SSH на нестандартном порту

───────────────────────────────────────────────
 Cloudflare API token для wildcard cert
───────────────────────────────────────────────

1. https://dash.cloudflare.com/profile/api-tokens → "Create Token"
2. Шаблон: "Edit zone DNS"
3. Permissions: Zone → DNS → Edit
4. Zone Resources: Include → Specific zone → vinnypuxtomoon.today
5. Continue → Create
6. Скопируй token (показывается ОДИН раз) — это твой $CF_Token

acme.sh внутри ноды:
  • Использует token для DNS-01 challenge (без портов открытых)
  • Выпускает cert на vinnypuxtomoon.today + *.vinnypuxtomoon.today
  • Cron ежедневно проверяет, обновляет за 30 дней до expiry
  • После обновления автоматически делает nginx -s reload

Если token утечёт — отозви в Cloudflare UI, выпусти новый.
Scope ограничен только DNS-зоной vinnypuxtomoon.today: атакующий
не получит доступ к остальной CF-инфраструктуре.

───────────────────────────────────────────────
 Что делает скрипт (5 фаз, до 1 ребута)
───────────────────────────────────────────────

1. Пакеты + обновление + ядро:
     apt install (gnupg, wget, curl, ufw, logrotate, git, openssl)
     apt upgrade
     XanMod (только если ядро < 6.12 и нет BBR3)
     ──► ОДИН ребут (если apt обновил ядро ИЛИ установлен XanMod)
2. Сетевая оптимизация (sysctl BBR3, conntrack, ulimits, fq qdisc)
3. Firewall (UFW: SSH, 443, 2222, 9100 + доп. порты по флагам)
4. Docker + контейнеры (параллельный pull, remnanode, node-exporter)
5. Клонирует репу с GitHub → собирает nginx + лендинг

С флагом --with-wl дополнительно в фазе 5:
  • UFW: открывает 80, 2083, 2085
  • docker-compose.yml: публикует :443 наружу
  • nginx.conf: раскомментирует TLS в WL-блоках
  • nginx/ssl/wl-ws: генерирует self-signed серт на IP ноды

На свежих дистрах (Ubuntu 25.04+, Debian 13+, kernel ≥ 6.12)
XanMod не ставится → 0 ребутов, установка ~2 мин.

После ребута systemd-сервис автоматически продолжит установку.
Если не сработал — просто запусти ту же команду повторно.

───────────────────────────────────────────────
 Логи установки
───────────────────────────────────────────────

# Полный лог (включая до и после ребута)
less /var/lib/node-setup/setup.log
tail -f /var/lib/node-setup/setup.log

# Логи resume-сервиса (только пост-ребут шаги)
journalctl -u node-setup-resume -e --no-pager
journalctl -u node-setup-resume -f

# Где скрипт остановился перед ребутом
cat /var/lib/node-setup/state.env

───────────────────────────────────────────────
 Сертификаты
───────────────────────────────────────────────

# Wildcard (selfsteal-ноды)
bash /opt/vinnypux-node/renew-cert.sh issue

# Принудительное обновление
bash /opt/vinnypux-node/renew-cert.sh install

# Self-signed для WL-нод генерируется автоматически флагом --with-wl.
# Ручная регенерация (если сменился IP):
IP=$(curl -s ifconfig.me)
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /opt/vinnypux-node/nginx/ssl/wl-ws/key.pem \
  -out    /opt/vinnypux-node/nginx/ssl/wl-ws/cert.pem \
  -subj "/CN=$IP" -addext "subjectAltName=IP:$IP"
cd /opt/vinnypux-node && docker compose restart nginx

───────────────────────────────────────────────
 Пересборка nginx после изменений
───────────────────────────────────────────────

cd /opt/vinnypux-node && docker compose down && docker compose up -d --build

# Или без даунтайма (только nginx reload):
docker compose exec nginx nginx -s reload
